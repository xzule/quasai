# Нефункциональные требования

NFR-001: Приватность данных. Система работает полностью локально. Сгенерированные тест-кейсы не передаются внешним сервисам. LLM-запросы выполняются в контуре пользователя.

NFR-002: Простота установки. Установка выполняется одной командой: ./run.sh или .\run.ps1. После клонирования репозитория пользователь сразу может запустить генерацию.

NFR-003: Кроссплатформенность. Система работает на Linux, macOS и Windows (через Docker).

NFR-004: Максимальный размер JSON. Выходной JSON-файл с box-сценариями не превышает 10 МБ. При превышении система выводит сообщение: "Result size exceeds 10 MB. Split the input requirements into multiple files and run generation for each separately".

NFR-005: Индикация прогресса. Система отображает прогресс выполнения в stdout на английском языке: "Parsing requirements" → "Generating: ... (X/Y)" → "Saving results" → "Results saved in:".
