# Функциональные требования

Чек-лист ревью — [RequirementsReview.md](RequirementsReview.md).

FR-001: Система принимает на вход Markdown-файл с требованиями и парсит его в структурированное представление (заголовки, секции, списки).

FR-002: Система передаёт структурированные требования в LLM и получает набор тестовых box-сценариев (позитивные, негативные, граничные).

FR-003: Система сериализует сгенерированные box-сценарии в JSON-файл и CSV-файл (разделитель ;, столбцы step0..stepN, expectedResult).

FR-004: Система предоставляет CLI-команду `generate <input.md>`. Результаты сохраняются в ./output/{filename}.json и ./output/{filename}.csv автоматически.

FR-005: Система собирается в Docker-образ локально через docker compose build.

FR-006: Репозиторий содержит скрипты run.sh (Linux/macOS) и run.ps1 (Windows), которые запускают генерацию одной командой и останавливают сервисы по завершению.
