# Форматирование документов

1. Общие принципы
1.1. Чисто, просто, понятно
1.2. Нет таблиц
1.3. Нет жирного шрифта
1.4. Нет курсива
1.5. Нет горизонтальных линий (---)

2. Заголовки
2.1. Документ начинается с # (одинарный хэш)
2.2. Подразделы выделяются только уровнем нумерации
2.3. Перед заголовком раздела — одна пустая строка, после заголовка — сразу содержимое (без пустой строки)
2.4. Заголовки в sentence case (не CAPS)

3. Общие документы (RFC-стиль)
3.1. Нумеруются только заголовки (1., 1.1., 1.1.1.)
3.2. Содержимое внутри секций — абзацы или списки
3.3. Нумерованные списки — только для дискретных элементов (требования, шаги)
3.4. Неупорядоченные списки — через дефис

4. Требования (IEEE-стиль)
4.1. Функциональные требования: FR-001, FR-002...
4.2. Нефункциональные требования: NFR-001, NFR-002...
4.3. Каждое требование — отдельный абзац с идентификатором

5. Имена файлов
5.1. Классический вид: Spec.md, Requirements.md, DataFlow.md
5.2. Не: SPEC.md, requirements.md, DATA_FLOW.md
