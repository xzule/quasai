# Ревью требований — чек-лист (ISTQB static testing)

1. Атрибуты качества
1.1. Корректность (correctness) — требование технически верно, реализуемо в рамках стека
1.2. Однозначность (unambiguity) — только одна интерпретация, нет расплывчатых формулировок
1.3. Полнота (completeness) — учтены все входы, состояния, граничные случаи
1.4. Непротиворечивость (consistency) — не конфликтует с другими FR/NFR и сценарием (Spec.md разд. 2)
1.5. Проверяемость (testability) — результат выполнения можно объективно проверить
1.6. Атомарность (atomicity) — одно требование описывает одну сущность/поведение
1.7. Трассируемость (traceability) — требование привязано к источнику (Spec.md, пользовательский сценарий)
