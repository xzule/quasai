# Инструкции проекта

1. Стиль кода
1.1. При работе с кодом форматировать в соответствии с docs/CodeFormatting.md
1.2. Создавать и/или редактировать код опираясь на тесты в docs/FunctionalRequirementsTests.md и docs/NonFunctionalRequirementsTests.md
1.3. После создания и/или редактирования кода выполнять пост-проверку по автоматизированному перечню тестов, описанному в docs/FunctionalRequirementsTests.md и docs/NonFunctionalRequirementsTests.md, и фиксировать результаты в этом же файле. Для новых требований — добавлять запись, для изменённых — обновлять существующую.

2. Коммуникация
2.1. Отвечать без преамбулы и постскриптума — сразу суть
2.2. Подтверждать перед удалением файлов, миграциями, коммитами

3. Описание проекта
3.1. При каждом turn читать Spec.md — epic проекта
3.2. Всю навигацию по проекту осуществлять через Spec.md и далее по ссылкам

4. Работа с .md файлами
4.1. Spec.md содержит markdown-ссылки `[title](path)` на все .md файлы, актуализировать ссылки
4.2. .md файлы хранятся в docs/
4.3. При работе с .md форматировать в соответствии с docs/DocsFormatting.md
4.4. Создавать и/или редактировать требования опираясь на чек-лист в docs/RequirementsReview.md
4.5. После создания и/или редактирования требований выполнять пост-проверку по чек-листу в docs/RequirementsReview.md и фиксировать результаты в этом же файле. Для новых требований — добавлять запись, для изменённых — обновлять существующую.
